# 📊 プロジェクト作品：クロスボーダーEC 日台市場売上分析システム (Cross-Border Sales Dashboard)

本プロジェクトは、バックエンドのデータベース構築、リレーショナルアーキテクチャの設計から、ビジネス意思決定に直結する高精度（High-Fidelity）なデータビジュアライゼーション（Figma）の実装までを網羅した、エンドツーエンド（End-to-End）のデータ分析システム設計です。2026年第2四半期（Q2）の実実績データに完全に準拠しています。

---

## 🛠️ フェーズ1：バックエンドデータベース開発 (PostgreSQL / DBeaver)

<img width="1470" height="956" alt="截圖 2026-07-20 中午12 37 27" src="https://github.com/user-attachments/assets/9133c8e4-3f5d-41af-92ae-0079a26be491" />

### 実装のポイント：
* **拡張性の高いリレーショナル設計**：PostgreSQL を用いてリレーショナルテーブル（`markets` と `monthly_sales`）を構築し、外部キー（FK）による厳格な整合性を担保。
* **財務レベルのデータ精度**：標準的な SQL 結合クエリ（`JOIN`）を作成し、`TO_CHAR(..., 'FM$999,999,999.00')` による財務フォーマット適用により桁あふれエラー（###）を防止。Q2総額 **$980,273.00** のクリーンなデータセットを抽出。

## 📐 フェーズ2：システムアーキテクチャおよびER図設計 (Draw.io)

<img width="612" height="369" alt="截圖 2026-07-22 上午10 07 26" src="https://github.com/user-attachments/assets/e53551aa-4c43-4f00-944b-9420bafad1c2" />

### 実装のポイント：
* **業界標準 ERD の作成**：実データベーススキーマに基づき、業界標準に準拠した **ERD (Entity Relationship Diagram)** を設計。
* **論理構造とカーディナリティの明記**：主キー（PK）と外部キー（FK）の伝播関係を明確にし、**1 : N（1対多）** のリレーションシップを正確に定義することで、構造化されたシステムデータフローの思考力を証明。

## 🎨 フェーズ3：高精度データビジュアライゼーション (Figma)

<img width="646" height="411" alt="截圖 2026-07-20 中午12 36 45" src="https://github.com/user-attachments/assets/36204de1-718a-449b-abc0-f4bac59c3287" />

### 実装のポイント：
* **データ完全準拠のビジュアル化**：チャート上のバーの高さ比率、カラーリング（赤：台湾 / 青：日本）、および上部のデータラベル（Data Labels）は、すべて PostgreSQL のクエリ結果と正確に一致。
* **主要なビジネスインサイト (Business Insights)**：
  * 4月および5月は日本市場がリードし、5月には四半期全体のピークを記録（日本 $245K、台湾 $210K）。
  * 6月は全体的に売上が減少したものの、**台湾市場（$142K）が日本市場（$100K）を逆転**し、意思決定者に強い対比印象を与えることに成功。
* **UIコンポーネント設計 (Auto Layout)**：Auto Layout を用いて、2色カラムとテキストラベルを垂直・水平方向にネストし、将来的なデータの増減に対して自動適応・崩れ防止（レスポンシブ対応）を実現。
